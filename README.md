## 작업 순서
1. 이슈 생성
2. 브랜치 생성 (ex. feat/#1)
3. add, commit, push
4. pr 생성 -> dev 브랜치로 (ex. [feat #1] 작업 내용)

<br/>

## Commit Convetion
ex. [feat #1] 회원가입
| 커밋 유형 | 의미 |
| --- | --- |
| feat | 새로운 기능 추가 |
| fix | 버그 수정 |
| refactor | 코드 리팩토링 |
| chore | 설정 파일, 패키지 매니저 수정 (Dockerfile, gradle, sh, yml) |
| style | 코드 formatting, 세미콜론 누락, 코드 자체의 변경이 없는 경우 |
| docs | 문서 수정 (md 파일) |

<br/>

## PR 리뷰
은향 -> 다인 -> 승연 -> 은진 -> 은향
<br/>
(리뷰 요청은 반대 방향)
